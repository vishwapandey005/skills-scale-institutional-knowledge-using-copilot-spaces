## Purpose
Provide a structured onboarding path for new team members joining OctoAcme projects. This checklist ensures consistent onboarding and reduces confusion for new joiners.

## For All Team Members

### First Day
- [ ] Complete HR and IT onboarding
- [ ] Access to communication channels (Slack, email lists, etc.)
- [ ] Introduction to immediate team members
- [ ] Access to project repositories and documentation
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review [Roles & Personas](octoacme-roles-and-personas.md)

### First Week
- [ ] Read all relevant process documents in the docs/ folder
- [ ] Attend team standup and weekly sync meetings
- [ ] Shadow a team member in your role
- [ ] Set up local development environment (if applicable)
- [ ] Review current sprint/iteration goals and backlog
- [ ] Meet with Project Manager for project context
- [ ] Meet with Product Manager to understand product vision

### First Month
- [ ] Complete first meaningful contribution (examples: merge a code change, update documentation, improve a process, resolve a customer issue, or complete a test plan)
- [ ] Participate in retrospective or planning meeting
- [ ] Understand escalation paths and communication patterns
- [ ] Build relationships with cross-functional team members
- [ ] Review project risk register and understand current risks
- [ ] Provide feedback on onboarding experience

## Role-Specific Onboarding

### Developers
- [ ] Review coding standards and contribution guidelines
- [ ] Set up CI/CD access and understand build pipeline
- [ ] Complete first code review
- [ ] Merge first pull request
- [ ] Review technical architecture documentation with Technical Architect
- [ ] Understand test coverage expectations with QA Lead

### Product Managers
- [ ] Review product roadmap and backlog
- [ ] Meet with key stakeholders
- [ ] Review customer research and feedback data
- [ ] Understand success metrics and measurement approach
- [ ] Shadow customer calls or support sessions with Support/Customer Advocate
- [ ] Review competitive analysis and market positioning

### Project Managers
- [ ] Review active projects and their status
- [ ] Understand resource allocation and capacity planning
- [ ] Review risk registers and past escalations
- [ ] Set up project tracking tools and dashboards
- [ ] Meet with cross-team dependencies
- [ ] Review communication cadence and stakeholder expectations

### QA Lead
- [ ] Review test strategy and coverage requirements
- [ ] Access to test environments and tools
- [ ] Review automated test suite and CI integration
- [ ] Understand quality gates and sign-off processes
- [ ] Review bug triage and prioritization process
- [ ] Meet with developers to understand testability practices

### Release Manager
- [ ] Access to deployment tools and production environments
- [ ] Review release schedule and windows
- [ ] Study deployment runbooks and rollback procedures
- [ ] Understand monitoring and observability setup
- [ ] Review past incidents and retrospectives
- [ ] Shadow a production deployment

### Technical Architect
- [ ] Review system architecture and design documentation
- [ ] Understand technical standards and patterns in use
- [ ] Review architectural decision records (ADRs)
- [ ] Meet with engineering leads across teams
- [ ] Understand technical debt backlog
- [ ] Review security and compliance requirements

### Support/Customer Advocate
- [ ] Access to support ticketing system
- [ ] Review customer communication templates
- [ ] Shadow support interactions and customer calls
- [ ] Understand escalation paths to engineering
- [ ] Review customer satisfaction metrics and trends
- [ ] Meet with Product Manager to understand customer segments

## Onboarding Feedback

At the end of your first month, please provide feedback on this checklist to your manager or Project Manager. Your input helps us continuously improve the onboarding experience.

### Feedback Questions
- What was most helpful during your onboarding?
- What was unclear or missing?
- What would you change about this checklist?
- How long did it take you to feel productive?
- What additional resources would have been helpful?
